Jelaskan dengan kemampuanmu apa itu event delegation dalam design pattern JavaScript. Tidak ada batas bobot kata dalam menjawab tugas ini, tetapi penilaian akan bergantung dari sepaham apa dan sebagus apa kamu menyajikan jawabanmu.

## Jawaban

Event Delegation itu teknik di JavaScript yang memungkinkan kita menangani banyak event hanya dengan satu event listener. Jadi daripada ngasih addEventListener ke setiap tombol atau elemen satu per satu, kita cukup pasang listener di elemen induknya (parent). Cara ini bisa dilakukan karena JavaScript punya mekanisme yang namanya event bubbling, yaitu saat sebuah elemen diklik, event-nya akan naik ke parent-nya. Dari parent tersebut, kita bisa mengecek elemen mana yang sebenarnya diklik menggunakan event.target.

Menurutku, keuntungan terbesar dari event delegation adalah bikin kode jadi lebih efisien dan rapi. Misalnya ada 100 tombol dalam sebuah halaman, kalau pakai cara biasa berarti harus membuat 100 event listener. Dengan event delegation, cukup satu listener di parent yang menampung semua tombol tersebut. Selain menghemat memori, cara ini juga memudahkan kalau nanti ada elemen baru yang ditambahkan secara otomatis melalui JavaScript karena event listener yang sudah ada tetap bisa menangani elemen baru tersebut tanpa perlu dibuat ulang.

Dalam pengembangan web, event delegation sering dipakai untuk menu navigasi, daftar tugas (to-do list), tabel data, atau daftar produk yang jumlah elemennya banyak dan bisa berubah-ubah. Karena semua logika event dikumpulkan di satu tempat, kode jadi lebih mudah dibaca dan dirawat. Walaupun ada beberapa jenis event yang tidak mendukung bubbling, secara umum event delegation adalah salah satu teknik yang sangat berguna untuk membuat aplikasi JavaScript yang lebih efisien dan mudah dikelola.
