## Soal

Dari dua kode di bawah ini, mana yang kamu ingin cari masalahnya dan perbaiki di tengah-tengah malam, katakanlah jam 1 malam? Mengapa?
```
function processUser(user) {
  if (user) {
    if (user.isActive) {
      if (user.hasPermission) {
        return doSomething(user)
      }
    }
  }
  return null
}
```
```
function processUser(user) {
  if (!isValidCandidate(user)) return null;
  return doSomething(user);
}

function isValidCandidate(user) {
  return user && user.isActive && user.hasPermission;
}
```

## Jawaban

Menurutku lebih baik pake yang code 2, karena mudah dibaca dan cepat dipahami, apalagi kalau harus cari problemnya di jam 1 malam pas kondisi lagi lelah.

Di code 1, pengecekannya harus mengecek satu per satu untuk perkondisian if nya, apalagi kalau if nya itu lebih banyak dari code 1 ini:
```
if (user) {
  if (user.isActive) {
    if (user.hasPermission) {
      return doSomething(user)
    }
  }
}
```
jadi kalau saya lagi ngantuk tuh, kode yang di atas ini lebih rawan bikin bingung. Ketika ada bug, saya juga harus beberapa kali akibat dari penggunaan if yang menumpuk

sedangkan di code 2, codenya mudah dipahami karena gak ruwet:
```
if (!isValidCandidate(user)) return null;
return doSomething(user);
```
Code yang di atas ini lebih enak dilihat karena dari awal sudah jelas kalau user tidak valid langsung berhenti. Kalau valid, baru lanjut menjalankan doSomething(user). Jadi proses membaca kodenya lebih ringan.

Selain itu, validasinya juga dipisahkan dengan fungsi sendiri:
```
function isValidCandidate(user) {
  return user && user.isActive && user.hasPermission;
}
```
jadi secara overral code yang kedua lebih enak dilihat dan lebih mudah untuk diotak atik, karena syarat user yang valid sudah dikumpulkan di satu tempat. Jadi kalau nanti ada tambahan kondisi, misalnya user harus sudah verifikasi email, tinggal ubah di fungsi isValidCandidate() saja, tidak perlu mengubah alur utama processUser() terlalu banyak.
