## Soal

Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.
```
const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}
```
```
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}
```
Menurutmu, kapankah kita saatnya menggunakan asersi atau eksepsi untuk fungsi seperti ini di atas? Apakah kita harus sepenuhnya asersi, atau sepenuhnya eksepsi? Lakukan riset dan berikan jawabannya dalam bentuk esai minimal 300 kata.

## Deskripsi

Kalau menurut aku, penggunaan asersi sama eksepsi itu sebenarnya punya fungsi yang beda, jadi gak bisa disamain atau dipilih salah satu aja. Keduanya sama-sama buat nge-handle error, tapi konteksnya beda.

Asersi biasanya dipakai buat ngecek hal-hal yang “harusnya gak mungkin salah” kalau program kita udah bener. Jadi lebih ke buat ngecek logika internal kita sendiri. Misalnya kita yakin suatu nilai pasti number, nah kita bisa pakai asersi buat mastiin itu. Kalau ternyata gagal, berarti ada bug di kode kita. Biasanya ini kepake pas development aja, dan bahkan di beberapa kondisi bisa dimatiin kalau udah masuk production.

Sedangkan eksepsi itu lebih ke error yang memang mungkin terjadi pas program dijalankan, apalagi yang datang dari user atau input luar. Contohnya kayak pembagian dengan nol atau input bukan angka. Nah ini bukan bug, tapi emang kemungkinan yang harus kita handle. Makanya pakai `throw` itu lebih cocok, karena bisa ditangkap pakai `try...catch` dan program tetap bisa jalan atau ngasih pesan error yang jelas.

Kalau dilihat dari contoh fungsi `divide`, menurut aku lebih cocok pakai eksepsi. Soalnya kondisi kayak `b === 0` atau input bukan number itu sangat mungkin terjadi di dunia nyata, apalagi kalau input dari user. Jadi harus ditangani dengan baik, bukan cuma dianggap kesalahan internal.

Tapi bukan berarti kita harus milih salah satu doang. Justru yang lebih bagus itu kombinasi keduanya. Asersi bisa dipakai buat ngecek asumsi internal biar kita cepat sadar kalau ada bug, sedangkan eksepsi dipakai buat handle error yang memang bisa terjadi saat program dipakai.

Jadi intinya, asersi itu buat “ngejaga logika kita sendiri”, sedangkan eksepsi itu buat “ngejaga program dari input atau kondisi luar yang gak valid”. Kalau dipakai bareng dengan tepat, hasilnya bakal lebih aman dan rapi.
